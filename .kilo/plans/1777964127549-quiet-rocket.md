# Translation Audit Plan: Restaurant Website

## Objective
Gather all section/page titles, subtitles, and small text above titles (descriptions) along with their page/section location to prepare for appropriate localization (context-aware translation rather than direct translation).

## Current State Analysis

### Project Structure
- **Framework**: Next.js App Router with internationalization support
- **Translation Pattern**: JSON locale files (`apps/web/locales/*.json`) loaded via `getTranslations()` function
- **Existing Locales**: English (en.json), Japanese (ja.json)
- **Data Source**: Restaurant-specific content stored in `data.json` files (static data, not translated via i18n)
- **UI Components**: Reusable components from `@workspace/ui` package with hardcoded English text

## Recommendations

1. **Extract ALL translatable strings** from code into the locale JSON structure
2. **Organize translations hierarchically** by component/page:
   - home (hero, about, menu, gallery, contact sections)
   - about (philosophy, team)
   - menu (badges, download)
   - contact (form details, labels)
   - brand (marketing materials)
   - common (navbar, footer, errors, placeholders)
3. **Add translation keys** for all hardcoded English strings in components
4. **Use interpolation** for dynamic values (restaurant name, phone) where appropriate
5. **Consider context** for terms like "Popular" (dish), "Veg" (vegetarian indicator)
6. **Keep form labels consistent** across pages
